### Fixed
- **Bug 1 ‚Äî currency no longer caches a failed registry lookup as a fresh fact** (bughunt 2026-06-13): an errored npm/PyPI lookup (network/HTTP/timeout, all collapsed to `undefined`) was unconditionally stamped `fetchedAt: today()`, so `isFresh()` reused it for the full 7-day TTL (never retried) and `audit-docs` read a `latest: undefined` entry as a present, current fact ‚Äî a fail-OPEN violation of the "inconclusive = FAIL" contract. The `resolve()` path now treats an unresolved lookup as not-a-fact: it carries forward the last-known-good value (with its prior date) or leaves `fetchedAt` empty, so `isFresh()` stays false (re-fetched next run) and `staleCurrency()` flags it (fail closed). `node:test`: `src/commands/currency.test.ts`.
- **Bug 2 ‚Äî daemon self-heal no longer clobbers a concurrent resume's record** (bughunt 2026-06-13): the orchestrator record has two unsynchronized writers (the daemon self-heal and the captain declare/resume path), and the heal wrote a whole record from a beat-start snapshot after a wide cmux window ‚Äî reverting a `sessionId` a resume wrote in between and re-arming the issue-#36 `--continue` fork hazard. `writeOrchestrator` now re-loads the on-disk record before writing and merges field-wise (the CLAUDE.md "re-load before mutating" guard), keeping whatever the resume wrote and only re-stamping the `surfaceId` the heal corrected. (A true cross-process concurrency test isn't feasible in `node:test`; `src/orchestrator-record.test.ts` is a focused test of the merge/re-stamp helper that stages the exact interleaving.)
- **Bug 3 ‚Äî `acquireSharedLock` no longer lets two daemons both win the single-instance lock** (bughunt 2026-06-13): the lock was created with O_EXCL and the pid written in a SEPARATE syscall, leaving an empty-file window; a racing daemon read `""` ‚Üí pid `undefined` ‚Üí skipped the live-owner guard ‚Üí deleted the in-progress lock and also won, double-starting the shared daemon. The pid is now created+written in one O_EXCL `writeFileSync({flag:"wx"})`, and an empty/partial pid makes the racer WAIT ‚Äî it only breaks a lock aged past `STALE_LOCK_MS` (a crashed holder), porting registry.ts's hardened guard. `node:test`: `src/daemon/config.test.ts` (fresh-empty preserved, stale-empty reclaimed, uncontended one-shot).
- **Bug 4 ‚Äî `probeStatus` no longer flips a LIVE worker to "dead" on a transient read error** (bughunt 2026-06-13): it caught ANY `readScreen` failure and returned `"dead"` with no not_found-vs-transient discrimination, so a busy-socket/EAGAIN hiccup dropped a running worker from the wave digest and let `watch --until-idle` declare quiescence early. It now fails closed to `"unknown"` (‚Üí KEEP, the durable-rescue branch takes over) and only returns `"dead"` when cmux's `not_found` code proves the surface is gone. The not_found discriminator `isGone` moved from `gc.ts` to `cmux.ts` so both paths share one tri-state check (re-exported from `gc.ts` for its tests). `node:test`: `src/status.test.ts` (transient‚Üíunknown, not_found‚Üídead, non-cmux‚Üíunknown, success‚Üíclassified).
- **Bug 5 ‚Äî `gc --apply` no longer phantom-deletes the legacy `~/.fleet/orchestrator.json` Captain record** (bughunt 2026-06-13): the backward-compat singleton has no dash, so it slipped past the `orchestrator-*` exclusion and `discoverSessions()` derived a phantom `"orchestrator"` session. Its own `session` field is the default (`"yoshi"`), so the Captain lookup missed ‚Üí `captain: "absent"` ‚Üí `planGc` said "remove" ‚Üí `--apply` `rmSync`'d a (possibly live) Captain's record. `discoverSessions()` now excludes the bare `orchestrator.json` (mirroring the `orchestrator-*.json` exclusion ‚Äî the dash was the discriminator that missed), and `"orchestrator"` is added to `RESERVED` so the file's "never enumerated as eligible" invariant holds by construction. `node:test`: `src/commands/gc.test.ts` (singleton excluded, ordinary sessions still discovered).

## 2026-06-11

### Added
- **`fleet spawn --done '<check>' [--max N]`** ‚Äî a stop condition on any spawn (closes the loop-engineering gap from `research/loop-engineering-vs-fleet-2026-06-11.md`: turns the eval gate from after-the-fact `fleet verify` into the loop itself). New pure decision core `src/done-loop.ts` (`node:test` coverage: gating, bounded re-dispatch vs exhaustion, feed-forward/escalation text); `spawn` records `doneCheck`/`doneMaxLoops`/`doneLoopCount` on the agent and the shared daemon drives it on stable-idle ‚Äî pass runs through the existing eval gate (`verify`, judge‚âÝgenerator, in the worker's dir) and auto-attaches the proof, fail re-dispatches the SAME worker (continued context) with the failure output bounded by `--max` (default 3), exhaustion sets a loud `fleet status` flag + urgent escalation and never auto-retries again. Registry/daemon-driven (shape (a)) so spawn keeps its fire-and-forget semantics; fail-closed (an unrunnable check is a FAIL) and never re-dispatches a `blocked-on-you`/`awaiting-input` worker (gated to `idle`). Spawn-side sibling of `fleet objective --done` (#33). (#34)
- **Contributor process** (first external issue/PR cycle, #30/#31): root `CONTRIBUTING.md` (dev setup ‚Äî `npm ci`, no build step, `npm run typecheck`, `npm test`; PR rules distilled from how #31 was handled ‚Äî surgical scope, CHANGELOG-under-Unreleased, verification evidence as commands+results, open-for-review ‚âÝ merge, judge ‚âÝ generator; issue + security guidance), `.github/` templates (`PULL_REQUEST_TEMPLATE.md`, `ISSUE_TEMPLATE/{bug_report,operational_review,config}` ‚Äî operational_review modeled on #30), `CODEOWNERS` (`* @dfeirstein`), a Node 24 `ci.yml` (`npm ci` ‚Üí typecheck ‚Üí test, job named `ci` for a required status check), and a README Contributing section. The process PR follows the process. (#32)
- **`fleet gc [--apply]`** ‚Äî sweep dead-session residue out of `~/.fleet` (closes #40, the manual 2026-06-11 cleanup gap): lists by default, removes with `--apply`, the registry/`.state.json`/capture-dir/daemon-`state-<s>.json` of every session with no live Captain AND no live worker. New pure decision core `src/commands/gc.ts` (`planGc`, `node:test` coverage: dead session, live Captain, live worker, unverifiable, mixed). **Fails closed at every check**: liveness uses error-distinguishing probes returning `present | absent | unknown` ‚Äî only cmux's `not_found` machine code reads as gone, any other failure is `unknown ‚Üí unverifiable ‚Üí KEEP`, so a transient cmux error mid-sweep can't delete a live session (a corrupt registry or malformed Captain record keeps too). Never enumerates `*.outcomes.jsonl`, `briefs/`, `browser-states/`, `worktrees/`, shared daemon files, or orchestrator records/prompts as eligible. (#46)
- **`install.sh` + self-updating `bin/fleet`** (closes #43): one curl-pipe-able installer (clones to `~/.local/share/fleet` or ff-pulls an existing checkout, `npm ci`, symlinks `fleet` + skill ‚Äî backing up a pre-existing real skill dir to a timestamped `.bak` ‚Äî then `fleet doctor`; idempotent, test-hookable via `FLEET_REPO`/`FLEET_BRANCH`/`FLEET_INSTALL_DIR`). `bin/fleet` gains a best-effort bash auto-update layer (the only pre-tsx hook): at most once/24h via a `~/.fleet/autoupdate-stamp`, only on a clean `main`, ff-only, bounded fetch (`GIT_HTTP_LOW_SPEED_*`), `npm ci` only when the lockfile moved, notices to stderr, `FLEET_NO_AUTOUPDATE=1` opt-out ‚Äî any failure degrades to running the command on existing code. New `fleet update` (explicit foreground pull: refuses on dirty/non-main, prints the CHANGELOG section-header delta, restarts the daemon if running) and a pure decision core `src/autoupdate.ts` (throttle/eligibility/lockfile-moved, `node:test` coverage) shared with `fleet update`. `fleet doctor` adds an install-mode report (checkout path, branch + short sha, ahead/behind `origin/main`, autoupdate on/off). README leads with the curl one-liner, manual clone kept as the developer path. (#47)

### Fixed
- **Captain resume safety** (#36, #37): a bare `fleet captain --resume` / `fleet orchestrate --resume` no longer defaults the name to "Captain" (which spawned a duplicate Captain that `claude --continue` forked into the live Captain's conversation) ‚Äî it now ERRORS and lists the live captains (records filtered by `surfaceExists`) with the exact `fleet captain <name> --resume` to run. On the captain/orchestrate path, `--help`/`-h` prints usage and any flag outside the known set (`--resume`/`--split`/`--no-daemon`/`--model`/`--command`/`--close-origin`/`--print`) errors with usage instead of spawning a stray Captain. Resume now targets `claude --resume <sessionId>` ‚Äî the id is resolved from the orchestrator record or the durable session map (`src/cmux-sessions.ts`, by the prior pane's surface) and stamped onto the record (best-effort: a fresh Captain's id is unknown at spawn, resolved on a later resume) ‚Äî falling back to `claude --continue` only with a loud fork-hazard warning when no id is resolvable. New `fleet captain <name> --resume --print` emits the in-pane manual relaunch recipe (`cd ‚Ä¶ && exec env FLEET_SESSION=‚Ä¶ claude --resume <id> --remote-control ‚Ä¶ --append-system-prompt-file ‚Ä¶`) without touching cmux. The greedy base parser is normalized for the captain path (`normalizeCaptainArgs`) so flag-before-name ordering (`fleet captain --resume yoshi`) reclaims the name instead of binding it as the flag's value ‚Äî which would otherwise slip past the no-name guard and spawn a stray Captain with resume silently dropped. Decision logic extracted to a pure `src/commands/captain-args.ts` with `node:test` coverage. (#44)
- **Daemon self-heals a stale Captain surfaceId instead of silently dropping a live Captain** (issue #39): an in-pane Captain relaunch (`Ctrl-C` ‚Üí `claude --resume`) changes the pane's surface UUID, so the orchestrator record's `surfaceId` went stale and the shared daemon's `surfaceExists` filter dropped a LIVE Captain (`watching 0 Captains`, no error). The beat now reconciles before dropping: when a record's surface is gone but its workspace still exists and the durable session map (`~/.cmuxterm/claude-hook-sessions.json`) shows exactly ONE live candidate surface in that workspace (excluding siblings' panes), it re-stamps the record's `surfaceId` (persisted, so every reader self-corrects) and keeps watching. Fail closed: an ambiguous match (>1 candidate) re-stamps nothing. A record that's neither live nor re-matchable for ‚â•2 consecutive beats fires ONE loud escalation before it stops being watched ‚Äî never silence. New pure decision core `src/daemon/selfheal.ts` (`node:test`: noop / clean re-match / ambiguous / workspace-gone), `writeOrchestrator` record writer, daemon `reconcileLiveCaptains` (impure orchestration seam-tested via injectable `ReconcileDeps`: sibling-pane exclusion, warn-once-at-2-beats, re-arm-on-vanish, persisted re-stamp). (#42)
- **`fleet send` no longer false-positives a steer swallowed by the boot splash** (issue #38): `submitToClaude` now gates on TUI readiness BEFORE typing ‚Äî a paste into a still-booting splash is silently eaten, and the cleared-input probe can't tell "text left the box" from "text never entered it". The readiness check is the SAME signal the spawn path waits on, now a pure classifier `classifyScreenReadiness` in `src/events.ts` (`ready | splash | unreadable`, `node:test` coverage for each) driven by an impure `waitForReady` poll in `src/cmux.ts` (matching spawn's 30s budget); `spawn.waitForClaudeReady` is refactored to share it (one heuristic, not two). A non-ready TUI yields a new `SubmitResult` `not-ready` (nothing typed) ‚Äî `fleet send` reverts its dispatch stamp and warns loudly ("worker TUI not ready ‚Äî steer NOT sent; check with `fleet read <id>` and re-send"), keeping the #31 failed/unverified contract intact. (#45)
- **Spawn/send no longer fail open after the TUI is ready** (the live "autostart silently dropped" failure, issue #30): `submitToClaude` now reports whether the text verifiably left the input box (`submitted | failed | unverified` ‚Äî read errors retry instead of silently passing, and only the final post-nudge read can verdict "failed"; a missing deciding read is "unverified"). `fleet spawn` marks a verified-failed brief `undispatched` (loud, sticky) instead of `running`; `fleet send` reverts its dispatch stamp and throws (warning that the steer is still in the box ‚Äî a blind retry would double-paste). SKILL.md + orchestrator-doctrine also gain a `fleet reply` permission protocol: in-scope actions stay autonomous (`allow`), destructive/critical actions are verify-then-decide (the Captain independently confirms target + goal-fit before answering; inconclusive = deny), standing grants (`always`/`bypass`) need an explicit user grant, and unattended waves negotiate the approval envelope up front. (#31)
- **`npm run typecheck` reproducible from a clean install**: `@types/node` is now a declared devDep (`^25.9.2`) ‚Äî tsconfig's `"types": ["node"]` previously resolved only where the package happened to be installed ad-hoc, so a fresh `npm ci` failed typecheck with TS2688. (#29)
